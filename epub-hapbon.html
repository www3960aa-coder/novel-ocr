📚 EPUB 합권/편집

여러 EPUB 파일을 하나로 합칩니다. 모든 처리는 브라우저에서만 이루어지며 서버로 전송되지 않습니다.

📖 사용 가이드

📁 1. EPUB 파일 업로드

  • 드래그&드롭 또는 클릭으로 여러 파일 한번에 업로드 가능
  • 아이폰: 영역 터치 → 사진/파일 앱에서 EPUB 선택
  • 파일 개수 제한 없음
  • 업로드 후 순서 조정 가능:
    • ↑ ↓ 버튼으로 한 칸씩 이동
    • 드래그로 자유 이동 (PC)
    • "이름순" 정렬: 1권, 2권... 사전순
    • "숫자순 자동 인식": 파일명의 마지막 숫자로 정렬 (권수가 끝에 있을 때)
    • ✕ 버튼: 개별 삭제

⚙️ 2. 합권 옵션

  • 계층 목차 적용: 각 권 아래 원래 소챕터 구조 유지 (권 → 챕터 → 챕터)
  • 단권별 목차 페이지 제거: 각 권 안의 toc.xhtml 같은 목차 페이지를 본문에서 제외 (전체 목차는 NCX로 통합)
  • 각 권 표지 모두 유지: 끄면 첫 번째 파일 표지만 남기고 나머지 권 표지 제거
  • 판권 페이지 제거: copyright, 저작권, 판권, ISBN, 무단전재 등 자동 감지해 제거
  • 각 권 속표지(intro) 제거: "~ N권" 같은 속표지 제거
  • EPUB 3.0 형식으로 저장: 최신 e북 리더에서 더 잘 작동. 끄면 EPUB 2.0 (호환성 우선).

📝 3. 메타데이터

  • EPUB 업로드 시 첫 파일에서 제목/작가/언어/발행처/출판일/설명 자동 추출 (원본에 있을 때만)
  • "제목 1권" 같은 권수 표시는 자동 제거됨 (수동 수정 가능)
  • 권 범위 표시: 파일명의 숫자로 자동 계산 (예: 1-4권, 1·3·5권)
  • 발행처/출판일/설명은 선택사항. 입력하면 EPUB 안 메타정보로 저장됨

📂 4. 파일명 형식

  • 자동 생성: [작가] 제목 권범위 完.epub
  • 예: [Lottoki] 로또 대박 기원 1-10권 完.epub
  • 제목/작가/권범위 바꾸면 자동 갱신
  • 직접 수정하면 자동 갱신 중단 (사용자 의도 우선)

📱 아이폰 사용 팁

  • Safari에서 이 HTML 파일 열기 (파일 앱에서 "Safari에서 열기")
  • 업로드: 드롭존 터치 → 파일 앱에서 EPUB 선택
  • 순서 조정: ↑ ↓ 버튼 사용 (드래그는 PC 전용)
  • 다운로드 후: "공유" 버튼 → "파일에 저장" 또는 다른 e북 앱으로 전송

🔒 개인정보 보호

  • 모든 처리는 브라우저 안에서만 이루어집니다
  • 파일이 서버로 전송되지 않습니다
  • JSZip 라이브러리 첫 로드 시에만 인터넷 필요 (CDN)

⚠️ 한계

  • 매우 큰 EPUB (수백 MB)은 브라우저 메모리 한계로 느릴 수 있음
  • 아이폰: 합계 100MB 이상 시 메모리 부족 가능
  • 판권 자동 인식이 100%는 아니라 가끔 다른 페이지가 잘릴 수 있음 → 결과 확인 권장
  • EPUB 2.0/3.0 모두 입력 가능, 출력 형식은 옵션으로 선택 가능
  • 복잡한 EPUB 자동 감지: CSS가 30개 이상이거나 서브폴더 구조면 디렉토리를 보존하여 CSS 깨짐 방지

판권 페이지 인식 기준:
파일명 (copyright, colophon, imprint, panskwon, rights 등) 또는 본문에서 다중 신호로 점수제 판정. 발행인/발행처/발행일 + ISBN + "무단 전재" 같은 한국 표준 판권 문구를 종합 평가합니다. 한자 표기(奧付, 版權)도 인식.

점수제 동작:
- 파일명에 명확한 단어 → 즉시 판권 판정
- 본문에 한자 판권 표기 → 즉시 판권 판정
- 한국 표준 문구 다수 발견 + 본문 길이 짧음 → 판권 판정
- 대화문 많음 / 본문 길이 김 → 일반 챕터로 판정 (감점)

속표지(intro) vs 목차 페이지 차이:
속표지는 표지 다음에 나오는 "제목 한 번 더 표시되는 페이지" (예: "로또 대박 기원 1권"). 합권 시 각 권 속표지가 다 들어가면 어색하므로 제거 권장. intro.xhtml 같은 파일.
목차 페이지는 본문 시작 전 챕터 목록 페이지. toc.xhtml, contents.xhtml, mokcha.xhtml. 합권 시 NCX 전체 목차로 통합되므로 단권 목차는 보통 제거.

속표지(intro) 인식 기준:
파일명에 intro가 들어가는 경우 자동 인식.
